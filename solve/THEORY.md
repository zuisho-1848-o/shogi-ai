# 詰将棋を「探索なし」で解く理論的アプローチ

一時メモ: 2026-06-18

---

## 核心的な問い

> 詰将棋を、探索をほぼせずに理論的に解決することはあり得るか？

**答え: 条件次第であり得る。**

---

## なぜ詰将棋は通常将棋より探索が少なくて済むか

詰将棋には通常将棋にない強い制約がある：

- **攻め方は必ず王手しなければならない** → 攻め方の合法手が通常の 1/10〜1/100
- **玉方は詰みを逃れる手のみ考えれば良い** → 守り側も選択肢が絞られる

この構造のおかげで df-pn 探索は数十手詰でもミリ秒で解ける。
「探索しているが事実上一瞬」という状態はすでに実現済み（既存技術）。

---

## 「探索ゼロ」が成立する3つの道

### 道1: 既知パターンへの帰着（最も現実的）

事前に「この形 = 詰み」という定理集を持っておき、
新しい局面が来たら**照合するだけ**にする。

```
例: 「玉が盤端 + 攻め方が金2枚を持つ形 → 必ず詰む」
    → この定理があれば、該当するか見るだけで即答
```

定理を作るときに探索は必要だが、**1回だけ**。
将棋の「手筋」は人間がこれを自然言語で行っているもの。

### 道2: 後退解析DBがそのまま「証明書」になる

後退解析で構築したDB（Phase 1 で作ったもの）は、
実質的に「全局面の詰み証明を事前に済ませた辞書」。

DBヒット = 探索ゼロ で答えが出る。

**課題**: 少数駒局面は現実的なDBサイズに収まるが、
駒数が増えると爆発する。→ 抽象化で解決したい（Phase 3）。

### 道3: パターンの合成による形式証明（研究フロンティア）

小さな詰みパターン A・B を合成して複雑な詰みパターン C を証明する、
という推論体系が作れれば、複雑な詰将棋を「定理の積み重ね」として解ける。

```
パターンA: 「玉が隅にいて金が利いていれば○手詰」
パターンB: 「玉が逃げると必ずパターンAの形になる」
合成     : → 元の局面は△手詰（証明完了）
```

これは**現時点で未解決の研究課題**。実現すれば将棋の形式証明システムになる。
Coq や Lean のような定理証明支援系との接続も視野に入る。

---

## 今のプロジェクト（Phase 1）との接続

```
Phase 1（完了）
  詰み局面 → 後退解析 → WIN/LOSS ラベルDB（3,430局面）
            ↓
Phase 2（次）: 少数駒局面（玉+N枚）を網羅的に後退解析
  → 「玉+金2枚」「玉+飛車」など駒種・枚数別の完全DB
            ↓
Phase 3: DBからパターンを自動抽出
  → 「○○の形は先手必勝」という命題を機械的に出力
  → 人間が読める定理集 = 「新しい将棋理論」
            ↓
Phase 4: パターン照合による探索ゼロ解決
  → 新局面が来たら定理集を引いて即答
  → 定理集にない場合は再帰的にパターン合成を試みる
```

---

## 現実的な目標設定

### 短期（Phase 2〜3）
- 「玉+駒3枚以下」の詰将棋を完全解決（DBとして）
- チェスの「6駒エンドゲームTB」に相当する将棋版

### 中期
- DBから抽出したパターンを自然言語で記述
- 「○○の形は後手必勝」という命題を自動生成

### 長期（研究的）
- パターン合成の推論体系を設計
- 「探索なしで証明できる詰将棋クラス」の境界を明らかにする

---

## 注意点・難しさ

- **抽象化の妥当性**: 「この駒は無関係」を証明するには局面を解く必要がある（循環）
- **持ち駒の爆発**: 将棋固有の持ち駒制度がDB・パターンを複雑にする
- **千日手**: 引き分けになる局面の扱いが形式化を難しくする
- **パターン合成の健全性**: 合成した証明が正しいことの保証が必要
